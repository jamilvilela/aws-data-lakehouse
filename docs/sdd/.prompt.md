## Project Summary

Production-ready **Data Lakehouse on AWS** implemented with Terraform, following the medallion architecture pattern. The infrastructure provisions S3 storage zones, IAM roles/groups/users, Lake Formation governance, and Glue Data Catalog resources.

**Repository:** `https://github.com/jamilvilela/aws-data-lakehouse`

## Key Architecture Decisions

1. **LF Roles as Principals** — IAM Groups cannot be used as Lake Formation principals. We use IAM roles per access level, with `sts:AssumeRole` from group policies. See [ADR-001](adr/ADR-001-lf-roles-as-principals.md).
2. **Separate S3 Buckets** — One bucket per medallion zone. See [ADR-002](adr/ADR-002-medallion-bucket-per-zone.md).
3. **Unified Analytics Role** — Single IAM role trusted by all analytics services. See [ADR-003](adr/ADR-003-single-analytics-role.md).

## Module Dependency Order

```
1. iam (roles first, no bucket dependencies)
2. s3 (buckets, no role dependencies)
3. lakeformation (depends on iam roles + s3 buckets)
4. data_catalog (depends on all above)
```

## Common Tasks

### Adding a New Glue Table
1. Add table name to `tables` variable in `infra/variables.tf` and `infra/tfvars/terraform.tfvars`
2. Create `infra/modules/data_catalog/tbl_<name>.tf`
3. Add LF permissions in `infra/modules/data_catalog/main_permisssions.tf`
4. Update `infra/modules/data_catalog/outputs.tf` if needed
5. Update `docs/sdd/feature.md` (F-004)

### Adding a New S3 Bucket
1. Add bucket name to `buckets` variable in `infra/variables.tf` and `infra/tfvars/terraform.tfvars`
2. Create `infra/modules/s3/main_bucket_<name>.tf`
3. Update `infra/modules/s3/outputs.tf`
4. Register as LF resource in `infra/modules/lakeformation/main_datalake_resources.tf`
5. Grant location permissions in `infra/modules/lakeformation/main_location_permissions.tf`

## Deployment

See `setup.sh` (deploy) and `destroy.sh` (teardown) at the repository root.

```bash
./setup.sh    # Deploy infrastructure
./destroy.sh  # Destroy infrastructure
```

Both scripts assume `datalake-admins-lf-role` before running Terraform.
